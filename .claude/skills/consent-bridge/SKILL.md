# consent-bridge — 근거 → 환자 설명·동의서 다리

위키에 **실제로 있는 논문의 수치**를, 환자가 알아듣는 말과 사전동의 문구로 번역한다.
`clinical-consult`가 술자용 근거 판단이라면, 이 스킬은 그 근거를 **환자에게 하는 말**로
바꾸는 마지막 한 걸음이다 — 근거는 같고, 청자가 환자다.

## ⛔ THE FOUR RULES (CLAUDE.md — 절대 위반 금지)

1. **웹검색 금지.** `WebSearch`/`WebFetch` 절대 사용 안 함. (PubMed MCP·PMID 조회는 별개로 허용되나, 설명의 근거는 오직 위키.)
2. **위키에서 먼저 답한다.** `sources/`·`wiki/`가 유일한 진실원천.
3. **위키가 부족하면 PDF 재독.** `papers/{stem}.pdf`를 pypdf로 더 파고, 위키를 갱신.
4. **논문이 없으면 없다고 말한다.** 지어낸 수치로 환자에게 하는 말을 만들지 않는다 — 환자 설명에서 수치 날조는 가장 위험한 위반.

환자에게 전달되는 문장일수록 근거 규율이 더 엄격하다. 훈련지식·직관으로 성공률·기간·위험 수치를 만들지 않는다.

## 언제 이 스킬이 아닌가 (경계)

- **술자용** 근거 판단·회색지대·"이 케이스 어떻게 할까" → `clinical-consult`
- 화난 환자 **민원·클레임 응대 작문** → `claim-coach`
- 논문을 위키에 **넣는** 작업 → `ingest-paper`
- 이 스킬은 **근거를 환자 눈높이 설명·동의서 문구로 번역**하는 작업에만.

(claim-coach는 *이미 벌어진 불만*을 진정시키는 사후 작문, consent-bridge는 *시술 전* 이해·동의를 만드는 사전 작문. 감정이 실린 항의면 claim-coach로 넘긴다.)

---

## Step 1 — 요청 파싱

세 가지를 분리해 명시한다:

- **환자의 질문 또는 설명 대상** — "왜 6개월 기다려요?", "임플란트 동의서", "발치 후 주의사항" 등. (필수)
- **환자/상황 인자** — 나이, 전신질환, 치아번호·부위, 술식 변수. 설명·수치를 바꾸는 modifier. (있으면)
- **전달 형태** — ①구두 설명 스크립트 ②서면 동의서 문구 ③둘 다. (기본: 둘 다)

⚠️ 환자 이름·차트번호 등 식별정보는 받지 않는다. 들어오면 빼라고 안내.

질문이 모호하면 가정을 명시하고 진행한다(chairside이므로 되묻기 최소화).

## Step 2 — 근거 검색 (QMD, 위키 한정)

`clinical-consult`와 동일한 검색 규율. **overview를 먼저** 노린다(warm start).

```bash
export PATH="/opt/homebrew/bin:$PATH"   # brew node 강제 (구 node ABI 회피)
cd /Users/oracleneo/llm-wiki
qmd search  "<정확 용어·시술·수치>"      # BM25 — 빠르고 안정적. 1차 경로.
qmd vsearch "<임상 개념 문장>"           # 벡터 유사도 — 개념 검색. LLM 없음, 빠름.
qmd get "wiki/<category>/<stem>.md"      # 히트 페이지 열람(줄번호)
```

- **1차 경로 = `qmd search`(BM25) + `qmd vsearch`(벡터).** 둘 다 LLM을 안 써서 헤드리스에서도 안 멈춘다.
- `qmd query`(하이브리드+LLM 리랭킹)는 비대화형 실행에서 멈출 수 있다 — 안 돌아오면 `search`/`vsearch`로 즉시 전환.
- 여러 각도로 2~3번: exact term(search) + 개념(vsearch) + 한국어 1회.
- 히트 중 **`wiki/overviews/` 우선**. 상위 후보를 `qmd get`/`Read`로 실제로 읽는다.
- 각 페이지의 **One-line Summary + Results 표 + `confidence:`** 확인. 환자 설명에 쓸 **핵심 수치를 페이지에서 그대로 발췌**해 둔다(성공률·생존율·기간·위험 %·N).

## Step 3 — Rule #4 게이트 (STOP 조건)

검색이 **주제상 관련 페이지를 못 내놓으면** 여기서 멈춘다.

> "이 주제 논문이 위키에 없어 근거 기반 환자 설명을 만들 수 없습니다 — PDF를 주시면 ingest 후 작성하겠습니다."

**일반 상식 수준의 환자 설명이라도 수치를 붙이려면 위키 근거가 있어야 한다.** 근거 없이 성공률·기간을 지어내 환자에게 말하지 않는다. 근거가 정성적(예: 기전 설명)만 있으면, 수치 없는 정성 설명만 하고 "구체 수치는 위키에 근거 없음"을 술자에게 알린다.

## Step 4 — 수치 번역 (근거 → 환자 언어)

발췌한 수치를 환자가 체감하는 형태로 **함께** 제시한다(원수치를 버리지 않는다):

- **비율 → 자연수 빈도.** "92%" → "10명 중 9명 정도". "3% 실패" → "100명 중 약 3명".
- **위험은 절대빈도로.** 상대위험(RR/OR)만 있으면 그대로 환자에게 말하지 말고, 가능하면 절대빈도로 환산해 설명하고 술자 트레일엔 원값을 남긴다. 환산 근거가 없으면 정성 표현("약간 높아짐")으로.
- **기간·과정은 "왜"와 함께.** "6개월"이 아니라 "뼈와 임플란트가 단단히 붙는 데 보통 N개월이 걸리기 때문".
- **불확실성 유지.** 범위(CI)·"연구마다 차이"·"환자마다 다름"을 없애지 않는다. 확정처럼 말하지 않는다.
- **전문용어 금지(환자 스크립트 한정).** 골유착·변연골소실 같은 용어는 풀어 쓴다("뼈에 붙는 것", "잇몸뼈가 조금 내려앉는 것"). 술자 트레일에서만 **한국어 (English, 약어)** 표기 유지.

## Step 5 — 출력 (3층 구조)

요청 형태에 맞춰 아래 중 필요한 층을 낸다. 기본은 3층 모두.

```
🗣️ 환자에게 (구두 스크립트)
  근거 기반 3문장 + 체감 수치. 중학생도 알아듣는 말. 존댓말 구어체.
  1문장: 무엇을/왜   2문장: 기대 효과(수치)   3문장: 정직한 한계·안심
  (환자가 이어서 물을 만한 질문 1~2개 + 한 줄 답을 덧붙일 수 있음)

📋 동의서 문구 (서면)
  격식체. informed consent 6요소를 담되 과장·보증 없이:
   ① 현재 상태  ② 제안 시술과 이유  ③ 기대 효과(수치)
   ④ 위험·합병증  ⑤ 대안(무치료 포함)  ⑥ 질문 기회 안내
  (필요 요소만. 대안·위험 누락 금지 — 동의서의 핵심.)

🔒 근거 트레일 (술자 전용 — 환자에게 주지 않음)
  • <환자에게 한 말> ← [[category/stem]] (confidence) · 원수치 그대로
  • 번역 로그: "92% → 10명 중 9명" 처럼 어떻게 환산했는지
  (환자 문장 하나하나가 어느 페이지·어느 수치에서 왔는지 술자가 검증 가능해야 함)
```

**인용·수치 규율(엄수):**
- 환자 스크립트·동의서의 **모든 수치는 실제로 읽은 페이지에서 발췌**. 기억으로 만들지 않는다.
- 근거 트레일의 `[[wikilink]]`는 **실제로 읽은 페이지만**. 인용을 지어내지 않는다.
- 페이지를 안 읽었으면 그 수치·주장을 환자 문장에 쓰지 않는다.

## Step 6 — 법적·윤리 가드 (반드시 준수)

환자에게 전달되는 문서이므로 claim-coach보다 사전동의 요건이 더 강하다:

- **보증·확정 금지.** "반드시 성공합니다", "100% 안전합니다", "절대 문제없습니다" 쓰지 않는다. 성공률은 실패 가능성과 함께 정직하게.
- **위험·합병증 명시(동의서 필수).** 흔한 위험 + 드물지만 중대한 위험을 누락하지 않는다. 근거에 있는 것만.
- **대안 명시(동의서 필수).** 무치료 시 경과 포함. 대안을 빼면 사전동의가 성립하지 않는다.
- **원인 단정·과실 표현 금지.** 예후·경과는 "보통 ~합니다", "~인 경우가 많습니다"로.
- **개별화 여지.** "환자분 상태에 따라 다를 수 있어 진료 시 확인이 필요합니다"를 남긴다.
- **면책 한 줄(항상, 술자용):** *환자 전달 전 술자가 사실관계·수치·개별 적용을 검토·수정하세요. 최종 책임은 술자에게 있습니다.*

## Step 7 — 지식 복리 (optional, 답변 뒤 한 줄)

- **논문 공백**이 드러나면: "이 부분은 위키에 논문이 없어요 — PDF 주시면 ingest."
- 자주 쓰는 설명이면: "이 설명 스크립트를 슬라이드/환자 안내물로 만들까요?"

## Worked example (형식 참고용)

**입력:** "임플란트 왜 6개월 기다려요? 환자한테 설명 스크립트랑 동의서 문구 둘 다."

**Step 1** — 대상: 임플란트 식립 후 보철까지 대기기간의 이유·기간을 환자에게 설명. 인자 미지정 → 일반 성인 가정. 형태: 둘 다.

**Step 2** — `qmd search "implant loading protocol healing time osseointegration"`,
`qmd vsearch "임플란트 식립 후 뼈에 붙는 기간과 부하 시점"` →
관련 overview/ISQ·loading 페이지 히트. `qmd get`으로 열어 대기기간·성공률·즉시부하 조건 수치 발췌.
(⚠️ 아래 수치·stem은 **형식 예시** — 실제 실행 시 반드시 검색·독해한 실 페이지·실 수치로 대체한다.)

**Step 5 출력 (형식만 참고):**

> 🗣️ **환자에게**
> "임플란트는 심고 나서 뼈와 나사가 단단히 하나로 붙는 시간이 필요한데, 보통 아래턱은 몇 달, 위턱은 조금 더 걸립니다. 이 기간을 충분히 기다린 경우 임플란트가 오래 유지되는 비율이 10명 중 9명 이상으로 높습니다. 급하게 이빨을 올리면 이 붙는 과정이 방해받을 수 있어서, 환자분 뼈 상태를 확인하며 안전한 시점에 진행하려는 것입니다."
> — 예상 추가질문: *"더 빨리는 안 되나요?"* → "뼈가 단단하고 조건이 맞으면 더 일찍 하는 방법도 있어, 상태 보고 판단드릴게요."
>
> 📋 **동의서 문구**
> "임플란트 식립 후 인공치아 장착까지는 임플란트와 턱뼈가 결합(골유착)하는 치유기간이 필요하며, 통상 [N]개월이 소요됩니다. 충분한 치유기간을 거친 경우 장기 유지율은 [원수치]%로 보고됩니다. 치유가 불충분한 상태에서 조기 부하 시 임플란트 실패 위험이 있습니다. 대안으로는 [조기부하 프로토콜/기존 보철] 등이 있으며, 각각의 장단점은 진료 시 설명드립니다. 개인의 골질·전신상태에 따라 기간과 예후가 달라질 수 있습니다. 궁금한 점은 언제든 문의하실 수 있습니다."
>
> 🔒 **근거 트레일 (술자용)**
> • 대기기간 [N]개월 ← [[implants/isq/<stem>]] (confidence) · 원문 수치 그대로
> • 장기 유지율 [원수치]% (95% CI …) ← [[implants/<stem>]] (sr+ma) · 번역: "→ 10명 중 9명 이상"
> • 조기부하 실패 위험 ← [[implants/isq/<stem>]] · 조건부(ISQ 임계값)
>
> *환자 전달 전 술자가 수치·개별 적용 검토·수정. 최종 책임은 술자.*

**교훈:** 환자에게 나가는 문장 하나하나가 근거 트레일의 실 페이지·실 수치로 되짚어져야 한다.
수치는 위키에서 발췌하고(Step 4에서 체감형으로 번역만), 없는 수치는 환자에게 말하지 않는다.
